# NEOCRAFT — Modern Voxel Sandbox

クールでモダンな雰囲気のマインクラフト風ボクセルサンドボックスゲーム。
ブラウザだけで動きます（Three.js / 単一HTMLファイル）。

## 遊び方

ローカルサーバーを立てて `index.html` を開きます。

```sh
python -m http.server 8765
# → http://localhost:8765 をブラウザで開く
```

## 操作

### スマホ・タブレット 📱

タッチ操作に対応しています。左下の仮想スティックで移動、画面ドラッグで視点操作、右下の ⚔️ で攻撃/破壊・🧱 で設置・⬆️ でジャンプ。上部のボタンで飛行切替・武器合成・ポーズ。ホットバーはタップで選択できます。

### PC（キーボード＋マウス）

| キー | 動作 |
|---|---|
| W A S D | 移動 |
| マウス | 視点 |
| Space | ジャンプ / （水中・飛行中）上昇 |
| Shift | ダッシュ |
| 左クリック | 攻撃 / ブロック破壊 |
| 右クリック | ブロック設置 |
| ホイール中クリック | ブロックスポイト |
| 1–9 / ホイール | ホットバー選択 |
| E | 武器の合成・装備メニュー |
| F | 飛行モード切替（C で下降） |
| Esc | メニュー |

## 特徴

- **テクスチャ付きの世界** — Canvasで自動生成した16×16ピクセルテクスチャ（草・石・木目・年輪・光る鉱石…）
- **プロシージャル地形生成** — ノイズベースの山・海・砂浜・雪山・森、地中にはグロウストーン鉱石
- **マグマの地下世界** — 地面を掘り進むと3Dノイズ生成の洞窟があり、深層（暗い深層石の大空洞）には溶岩の湖が広がる。溶岩は触れるとダメージ、深いほど暗くなり鉱石と溶岩の光が頼り
- **ザコ敵** — 地上には「スライム」「シャドウ」、マグマエリアには溶岩上を跳ねる「マグマスライム」・急襲してくる「ファイアバット」・ヒビが光る重量級「シンダーゴーレム」。HPバー・ノックバック・撃破ドロップ付き
- **戦闘システム** — ライフ10ハート、敵の接触ダメージ、被弾フラッシュ、自然回復、死亡＆リスポーン
- **武器の合成と装備** — 素材（木材・石・グロウストーン・闇のコア）を集めて「木の剣 → 石の剣 → ネオンブレード」を合成。装備した武器が手元に表示され、攻撃力が変わる
- **ボス「古代ゴーレム」** — 地上に1体だけ佇む巨大ゴーレム。普段は大人しいが、攻撃すると激怒して襲ってくる。討伐すると豪華ドロップ
- **ボス「マグマドラゴン」** — 地下の溶岩湖を旋回する翼幅8ブロックの巨竜（HP80）。手を出さなければ襲ってこないが、攻撃すると激怒して追い回し、火球を吐いてくる
- **ボス「黒龍」** — 地上の上空を旋回する黒紫の巨竜（HP120）。攻撃するまで襲ってこない。激怒すると追跡＋火球
- **ボス「氷雪のゴーレム」** — 雪山の頂上に佇む氷のイエティ（HP100）。攻撃するまで襲ってこない
- **共有ワールドボス** — マルチ接続時はボスのHP・激怒・撃破・復活が全員で同期され、みんなで同じボスを削る共闘レイドになる（討伐で全員に報酬、30秒後に復活）
- **ボス討伐ランキング** — Rキー（スマホは🏆ボタン）でオンラインの仲間のボス討伐数ランキングを表示。討伐数はセーブに永続保存され、近くで戦った参加者にカウントされる
- **スマホ対応** — 仮想スティック＋タッチボタンでブラウザからそのままプレイ可能
- **昼夜サイクル** — 朝焼け・夕暮れ・星空がシームレスに変化（1周4分）
- **モダンなビジュアル** — アンビエントオクルージョン、ACESトーンマッピング、フォグ、流れる雲、水面アニメーション
- **ガラスモーフィズムHUD** — ホットバー、ハート、装備チップ、合成パネル、トースト通知
- **長押し採掘** — ブロックごとの硬さ＋円形ゲージ。つるはし/斧/シャベルで適正ブロックが3.5倍速
- **クラフト（データ駆動レシピ）** — 武器・道具・防具・建材・食料を統一テーブルで管理。**作業台**を置くと高度なレシピが解放。板材・石レンガ・松明・かまど
- **階段・ハーフブロック** — 板材/石レンガの階段（向き自動）とハーフ。半ブロック段差は歩いたまま登れる
- **防具** — 石の兜・鎧・脚甲（1部位ごとに被ダメージ-15%）。さらに闇のコアで作る**黒龍の鎧**（-50%・装備中は黒炎をまとう）。軽減は合計最大80%
- **動物と食料** — 草原を歩き回る**ブタ**を狩って生肉を入手、**かまど**の近くで焼き肉に調理（Qキーで食べてHP+6回復）
- **松明ライティング** — 置いた松明が周囲7ブロックを照らす（暗い地下坑道で効果大）
- **セーブ/ロード** — 「シード＋差分」方式の自動セーブ（10秒ごと＋タブ離脱時）。`?seed=xxx` で別ワールド生成
- **マルチプレイ布石** — 全ワールド変更がアクション（applyAction）経由、同期アダプタの差し込み口、リモートプレイヤー描画土台まで実装済み（Firebase接続は今後）

## マルチプレイのセットアップ（Firebase）

ゲーム本体はGitHub Pagesのまま、同期だけFirebase Realtime Database（無料Sparkプラン）に任せる構成。
専用サーバーは不要です。

1. [Firebaseコンソール](https://console.firebase.google.com/)でプロジェクトを作成
2. **構築 → Realtime Database → データベースを作成**（ロケーションは asia-southeast1 など）
3. 「ルール」タブに下記のセキュリティルールを貼り付けて公開
4. **構築 → Authentication → ログイン方法 → 匿名** を有効化
5. **プロジェクト設定（⚙）→ マイアプリ → ウェブアプリを追加**（ニックネーム任意、Hostingは不要）
6. 表示された `firebaseConfig` を `index.html` 内の `FIREBASE_CONFIG = null` の部分に貼り付け
   （`databaseURL` が含まれていることを確認。なければRealtime DatabaseのURLを追記）
7. デプロイすれば、**同じURL・同じシードを開いた人同士が同じワールドに入れる**
   （初回にプレイヤー名を聞かれます。切断すると自動で退室扱いになります）

セキュリティルール：

```json
{
  "rules": {
    "players": {
      "$uid": { ".read": true, ".write": "auth.uid === $uid" }
    },
    "worlds": {
      "$world": {
        "seed":    { ".read": true, ".write": "auth != null" },
        "changes": { ".read": true, ".write": "auth != null" },
        "bosses":  { ".read": true, ".write": "auth != null" },
        "chat":    { ".read": true, ".write": "auth != null" },
        "beacon":  { ".read": true, ".write": "auth != null" },
        "scores":  {
          ".read": true,
          "$uid": { ".write": "auth.uid === $uid" }
        },
        "online":  {
          ".read": true,
          "$uid": { ".write": "auth.uid === $uid" }
        }
      }
    }
  }
}
```

データ構造（シード＋差分方式。マップ全体は保存しない）：

```
worlds/
└── world_{seed}/
    ├── seed                ← マップ再生成用
    ├── changes/            ← 弄ったブロックの差分だけ "x_y_z": blockId（0=壊した）
    └── online/{uid}        ← 今いる人 { name, x, y, z, yaw }（切断で自動削除）
```
